Release Notes


Version 1.5.8 (1134)

  • Computer Associates eTrust Antivirus 7 is now supported directly.
    If you are running eTrust 7.1 on your NetWare Server, MailFilter can now use it without waiting for the real-time scanner.
    Make sure eTrust is loaded and running before starting MailFilter.
    (Not available with the legacy MailFilter.)
    Please have a look below for configuration details.
  • ZIP files will be unpacked
    If you have the AV module licensed, MailFilter will try to unpack any .ZIP attachments and scan the archive contents.
    You can also drop encrypted ZIPs or ZIPs that contain special file names (works the same as with normal e-mail attachments).
    See below for correct configuration.
  • Configuration Editor (MFConfig.nlm) has been merged into MailFilter.
    Online Configuration: Press F10 in MailFilter main screen
    Offline Configuration: load mailflt -t config
  • Mail Restore (mfrest.nlm) has been merged into MailFilter.
    Online Restore: Press F8 in MailFilter main screen
    Offline Resore: load mailflt -t restore
  • NRM snap-in has been merged into MailFilter.
    You can enable the Netware Remote Manager snap-in from the Configuration Editor.
  • Graphical Filter Editor for Windows included.
  • New Installation and Upgrade Wizard
    MailFilter can be installed on another volume as SYS:
    MailFilter can be installed in a seperate directory (useful for Novell Cluster Services installations)
    Does not require a drive mapping to SYS:
  • MailFilter can now log in to the NetWare server, which may be required for some AV Engines to see the file moves.
    (Not available with the legacy MailFilter.)
  • Multi2One now also changes the Return-Path header generated by newer GWIA Versions.
  • Updated MFFilterICE to match new configuration format.
  • "Legacy" MailFilter has been renamed to MFLT50.NLM. Only the LibC Version is named MAILFLT.NLM now.
  • Troubleshooting "Cannot communicate with NWSNUT!" Error:
    This sometimes happens on NetWare Versions prior to Consolidated Support Pack 11 and when MailFilter is loaded protected.
    If you get this error, try loading THREADS.NLM into the MailFilter address space (before loading MailFlt.NLM!)
    If you can't resolve it this way, please contact MailFilter support.

Recommended .ZIP match rules

Field: Archive Content: Filename
Action: Drop Mail
Matchtype: Match
Expression: <encrypted file>
Description: ZIP encrypted file
Rule Enabled Incoming: Yes
Rule Enabled Outgoing: No
Field: Archive Content: Filename
Action: Drop Mail
Matchtype: Match
Expression: \.(exe|scr|pif|com|vbs|lnk)
Description: ZIP with executables
Rule Enabled Incoming: Yes
Rule Enabled Outgoing: No

Computer Associates eTrust AV Setup

eTrust 7.1 must be loaded and running before MailFilter starts. While MailFilter is running, you cannot unload eTrust completely.

MailFilter configuration:
  • Enable "Unpack Mails for Virus Scanner"
  • Remove your "Login as Username/Password" settings
  • Add the following rule:
    Field: Virus Scan
    Action: Drop Mail
    Matchtype: Match
    Expression: - (this is filled in automatically)
    Description: Virus Scan failed. (gets overridden if AV NLM returns a Virus Name)
    Rule Enabled Incoming: Yes
    Rule Enabled Outgoing: Yes

eTrust Configuration:
  • Exclude the GWIAHOME\MF\MFSCAN directories from the Real Time Scanner.
  • Make sure eTrust 7.1 is loaded and running before MailFilter gets loaded.

If you have any questions, please contact MailFilter Support!
